# v1.3.2 — 补全行为修复

## 背景

v1.3.0 引入了 `replaceMode` 属性，支持 `line`/`file` 级别的替换范围。上线后发现两个问题：

1. `lineText` 环境变量在 dot 不在行尾时，无法正确移除触发用的 `.`
2. `replaceMode` 为 `line`/`file` 时，直接设置 `item.range` 为大范围会导致 VS Code 补全列表过滤异常（因为 VS Code 会用 `item.range` 内的文本做前缀匹配）

## 问题详情

### 问题 1：lineText 中的 dot 未正确移除

旧逻辑使用 `lineText.replace(/\.$/, '')` 移除行尾的 `.`，但当用户在行中间输入 `word.` 时（例如 `abc.def`），行尾并没有 `.`，导致 `lineText` 中仍包含触发符。

### 问题 2：replaceMode 补全过滤异常

旧逻辑在 `line`/`file` 模式下将 `item.range` 设为整行/整文件范围。VS Code 会使用该范围内的文本作为过滤前缀，导致补全项被错误过滤掉或排序异常。

## 方案

### 修复 1：精确移除 dot

```typescript
// 旧：只移除行尾 dot
lineText: lineText.replace(/\.$/, ''),

// 新：精确移除光标位置的 dot
lineText: lineText.slice(0, position.character - 1) + lineText.slice(position.character),
```

### 修复 2：additionalTextEdits 替代大范围 range

将 `item.range` 始终设为 `word.` 的精确范围（保证 VS Code 正常过滤），`line`/`file` 模式下通过 `item.additionalTextEdits` 删除 word 范围之外的内容：

```typescript
// item.range 始终只覆盖 word. 部分
item.range = wordRange;

// line/file 模式：用 additionalTextEdits 删除额外内容
if (replaceMode === 'line' || replaceMode === 'file') {
    const edits: vscode.TextEdit[] = [];
    if (fullRange.start.isBefore(wordRange.start)) {
        edits.push(vscode.TextEdit.delete(new vscode.Range(fullRange.start, wordRange.start)));
    }
    if (position.isBefore(fullRange.end)) {
        edits.push(vscode.TextEdit.delete(new vscode.Range(position, fullRange.end)));
    }
    item.additionalTextEdits = edits;
}
```

### 修复 3：InnerRule.replaceMode 必选化

`InnerRule` 中 `replaceMode` 从可选改为必选，默认值 `'word'` 在 `ConfigCache` 构建时统一赋值，避免下游需要重复处理 `undefined`。

## 变更文件

| 文件               | 变更内容                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| `src/extension.ts` | `lineText` 精确移除 dot；`replaceMode` 改用 `additionalTextEdits` 实现  |
| `src/lib.ts`       | `ConfigCache` 中为 `replaceMode` 赋默认值 `'word'`                      |
| `src/types.ts`     | `InnerRule.replaceMode` 改为必选字段                                     |

## 兼容性

- 无配置格式变更，用户无需修改任何规则
- `replaceMode` 默认值仍为 `'word'`，行为与旧版一致
