# v1.3.0 — replaceMode：替换范围控制

## 背景

在此前的版本中，用户触发补全后，扩展仅会替换光标前最近的 `word`（即 `.` 之前的单词）。例如：

```
输入：abc def.
选择 toUpperCase
结果：abc DEF  ← 只有 def 被替换
```

当用户希望对整行或整个文件应用格式转换时，无法通过规则配置实现，只能手动操作。

## 需求

在 rule 配置中增加 `replaceMode` 属性，让用户可以控制补全结果的替换范围。

## 设计

### 新增属性

| 属性          | 类型                       | 必填 | 默认值 | 说明                         |
| ------------- | -------------------------- | ---- | ------ | ---------------------------- |
| `replaceMode` | `word` \| `line` \| `file` | 否   | `word` | 控制补全结果替换的文本范围   |

### 行为说明

| 值     | 替换范围           | 示例（输入 `abc def.`，规则结果为 `DEF`） |
| ------ | ------------------ | ----------------------------------------- |
| `word` | 仅替换最近的单词   | `abc DEF`                                 |
| `line` | 替换当前整行       | `DEF`（整行被替换为格式化结果）           |
| `file` | 替换当前整个文件   | 整个文件内容被替换为格式化结果            |

> 格式化逻辑（snippet / function 规则）本身不变，仅替换范围不同。

## 实现

### 变更文件

| 文件               | 变更内容                                                           |
| ------------------ | ------------------------------------------------------------------ |
| `src/types.ts`     | `Rule` 接口新增 `replaceMode?: 'word' \| 'line' \| 'file'`        |
| `src/extension.ts` | `provideCompletionItems` 中根据 `rule.replaceMode` 计算替换范围   |
| `package.json`     | `dot-anything.rules` JSON Schema 新增 `replaceMode` 枚举属性       |

### 核心逻辑（extension.ts）

```typescript
const replaceMode = rule.replaceMode ?? 'word';
let replaceRange: vscode.Range;

if (replaceMode === 'line') {
    replaceRange = document.lineAt(position.line).range;
} else if (replaceMode === 'file') {
    replaceRange = new vscode.Range(
        new vscode.Position(0, 0),
        document.lineAt(document.lineCount - 1).range.end,
    );
} else {
    // 'word'（默认）
    replaceRange = new vscode.Range(
        new vscode.Position(position.line, startChar),
        position,
    );
}
item.range = replaceRange;
```

## 使用示例

### line 模式：格式化整行注释

```json
{
    "trigger": "comment",
    "description": "将整行内容转为注释",
    "replaceMode": "line",
    "snippet": "// #lineText#"
}
```

输入 `abc def.` → 选择 `comment` → 整行变为 `// abc def`

### file 模式：全文大写（适用于纯文本处理场景）

```json
{
    "trigger": "upper",
    "description": "全文转大写",
    "type": "function",
    "replaceMode": "file",
    "snippet": "(env) => env.lineText.toUpperCase()"
}
```

> 注意：`file` 模式会替换整个文件内容，请谨慎使用。

## 兼容性

- 默认值为 `'word'`，与旧版行为完全一致，无破坏性变更
- 已有规则无需修改
