# v1.5.0 — 欢迎页、国际化与默认规则

## 背景

扩展此前无首次安装引导，用户需要自行查阅文档了解配置方式。同时配置项描述仅有英文，对中文用户不友好；`dot-anything.rules` 默认为空数组，新用户开箱即用体验差。

## 需求

1. 首次安装时展示欢迎页，引导用户了解核心功能和配置方式
2. 扩展配置项和 UI 文案支持中英文国际化
3. 提供开箱即用的默认规则（`log` 和 `//`）

## 设计

### Welcome 欢迎页

- 首次安装时自动弹出 WebView 面板，展示 logo、功能说明、demo GIF、默认规则配置和文档链接
- 使用 `globalState` 记录是否已展示，确保只弹出一次
- debug 模式下每次激活都会展示，便于开发调试
- 支持 `vscode.l10n` 实现运行时多语言

### 国际化（i18n）

- `package.json` 中所有用户可见描述改为 `%key%` 占位符
- `package.nls.json`（英文）、`package.nls.zh-CN.json`（中文）提供配置项翻译
- `l10n/bundle.l10n.zh-CN.json` 提供运行时 UI 翻译（欢迎页文案）
- `package.json` 新增 `"l10n": "./l10n"` 配置

### 默认规则

- `dot-anything.rules` 默认值从 `[]` 改为包含两条规则：
  - `log`：快速插入 `console.log`（含文件路径、行号、变量信息）
  - `//`：将当前行转为单行注释
- 新增 `doc/rules/default.md` 文档，方便用户在自定义后找回默认规则

## 使用示例

### 欢迎页

首次安装后自动弹出：

- 展示 logo 和简介
- 演示 `word.log` 和 `.//` 两个默认规则的用法（含 GIF 动图）
- 提供"查看文档"和"打开设置"快捷按钮

### 默认规则

安装即可使用，无需配置：

```
name.log  →  console.log('🖨️ /src/demo.ts[10:5] name:', name);
.//       →  // 当前行内容
```

## 兼容性

- 欢迎页仅首次安装弹出，不影响现有用户
- 国际化为纯 UI 文案变更，不影响功能行为
- 默认规则变更：**若用户已在 settings.json 中自定义 `dot-anything.rules`，则不受影响**（VS Code 以用户设置优先）。仅影响从未配置过的新用户
