# v1.4.0 — per-rule `pattern` 正则匹配 + `match` 环境变量

## 背景

在此前的版本中，扩展硬编码了 `/(\S+)\.$/` 正则来提取 `.` 前的 word。这意味着：

1. 用户**必须**在 `.` 前输入至少一个非空字符才能触发补全
2. 所有规则共享同一个匹配逻辑，无法按规则自定义触发条件
3. 无法获取正则中的多个捕获组

例如，对于 "comment"（注释当前行）这类不需要 word 的规则，用户被迫输入一个无意义的单词才能触发。

## 需求

1. 每条规则可配置 `pattern` 正则，控制何时显示该规则的建议
2. 新增 `match` 环境变量，暴露正则匹配的所有捕获组
3. 文本模式中对数组类型环境变量通用支持索引访问和逐元素格式化

## 设计

### 新增属性

| 属性      | 类型   | 必填 | 默认值       | 说明                                     |
| --------- | ------ | ---- | ------------ | ---------------------------------------- |
| `pattern` | string | 否   | `(\S+)$`     | 正则表达式，匹配光标前文本（末尾 `.` 已去除） |

### 新增环境变量

| 环境变量    | 类型       | 说明                                                  |
| ----------- | ---------- | ----------------------------------------------------- |
| `match`     | `string[]` | 正则匹配所有组：`match[0]`=完整匹配, `match[1]`=第一个捕获组(=word), ... |

### 文本模式中 match 的访问方式

| 语法              | 说明                                | 示例                    |
| ----------------- | ----------------------------------- | ----------------------- |
| `#match#`         | 所有匹配组拼接                      | `match.join('')`        |
| `#match.N#`       | 指定捕获组（N 为索引）              | `#match.0#` = 完整匹配  |
| `#match^format#`  | 对每个元素应用格式函数后拼接        | 每个组都会被格式化      |
| `#match.N^format#`| 对指定捕获组应用格式函数            | `#match.1^toUpperCase#` |

> 以上规则对所有数组类型的环境变量通用，不仅限于 `match`。

### 行为说明

- 未配置 `pattern` 的规则使用默认 `(\S+)$`，行为与旧版完全一致
- `word` 环境变量取自 `ruleMatch[1]`（第一个捕获组），无捕获组时为空字符串
- 无效正则会在输出面板打印错误日志，该规则被跳过

## 实现

### 变更文件

| 文件               | 变更内容                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| `src/types.ts`     | `Rule` 新增 `pattern?`，`InnerRule` 新增 `patternRegex`，`EnvVars` 新增 `match` |
| `src/lib.ts`       | `getRules()` 编译 pattern 为 RegExp；`applyFormat()` 通用数组值替换逻辑 |
| `src/extension.ts` | 删除全局正则匹配，改为 per-rule patternRegex 匹配                       |
| `package.json`     | rules schema 新增 `pattern` 字段，版本号升至 1.4.0                      |

## 使用示例

### 无 word 触发（如注释当前行）

```json
{
    "trigger": "//",
    "description": "注释当前行",
    "pattern": "",
    "replaceMode": "line",
    "snippet": "// #lineText#"
}
```

只需输入 `.` 即可触发，无需在 `.` 前输入任何单词。

### 仅匹配数字

```json
{
    "trigger": "px",
    "description": "数字转 px",
    "pattern": "(\\d+)$",
    "snippet": "#word#px"
}
```

输入 `16.` → 选择 `px` → 得到 `16px`。非数字输入不会触发此规则。

### 使用多捕获组

```json
{
    "trigger": "swap",
    "description": "交换两个单词",
    "pattern": "(\\w+)\\s+(\\w+)$",
    "snippet": "#match.2# #match.1#"
}
```

输入 `hello world.` → 选择 `swap` → 得到 `world hello`

### 函数模式访问 match

```json
{
    "trigger": "info",
    "type": "function",
    "pattern": "(\\w+):(\\d+)$",
    "snippet": "(env) => `file: ${env.match[1]}, line: ${env.match[2]}`"
}
```

输入 `main:42.` → 得到 `file: main, line: 42`

## 兼容性

- 默认 pattern 为 `(\S+)$`，与旧版行为完全一致，无破坏性变更
- 已有规则无需修改
- `word` 环境变量仍然可用，始终等于第一个捕获组
